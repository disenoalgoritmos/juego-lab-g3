ALGORITMO FINALIZAR PARTIDAS


- Cuando aparezca un mensaje FINISH, si el oponente vuelve a responder con otro, se termina la partida resultado ganador el primer jugador 
que mandó uno de ellos.

- Cuando aparezca un mensaje ERROR, si el oponente vuelve a mandar el mismo sucesor (hay que tenerlo almacenado por si acaso), se termina la 
partida resultando ambos jugadores perdedores. ESTO POR AHORA NO ESTÁ IMPLEMENTADO

- Cuando aparezca un mensaje ERROR, si el oponente responde con otro sucesor, simplemente se aumentará en una unidad un contador de fallos 
para cada jugador. El primer jugador que llegue a 3 fallos, se considerarán ambos jugadores como perdedores, finalizando así la partida.

- En todo caso, el bucle de los clientes sólo parará cuando reciban por parte del servidor game el mensaje BYE.

- El servidor game no reenviará el mensaje FINISH, sino que cuando reciba uno de ellos, aumentará en uno el contador de mensajes FINISH 
y pasará al oponente el sucesor que se supone que lo hará perder. En caso de que envíe un mensaje FINISH, cortará la partida 
(en caso contrario se sigue jugando normalmente, se resetea el contador de mensajes FINISH ¿debería sumarse un error al juagdor oponente?)

- Cuando ambos jugadores pierden, la variable ganador del servidor será None
